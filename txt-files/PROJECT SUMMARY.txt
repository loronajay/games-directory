PROJECT SUMMARY
================================================
JAY'S RETRO ARCADE – PROJECT ARCHITECTURE SUMMARY (UPDATED)
PROJECT VISION

This project is not just a website. It is intended to become a complete retro gaming platform that can run:

• As a public website
• As a local offline arcade
• As a Raspberry Pi arcade cabinet
• Potentially as a standalone retro gaming system

The website currently being built is effectively the operating system for the arcade interface.

The long-term goal is that this same system will eventually run on a physical arcade cabinet connected to a display and arcade controls via a Raspberry Pi.

The web version is the first iteration of the arcade frontend.

PLATFORM STRUCTURE

The system is organized into three main layers.

ARCADE INTERFACE (WEBSITE UI)

This layer controls the user interface of the arcade.

Responsibilities include:

• Menu navigation
• Game selection
• Preview videos
• UI animations
• CRT visual effects
• Gamepad / keyboard navigation
• Mobile scaling
• Audio effects
• Play count display

Main pages include:

INDEX PAGE

Mode selection screen.

Choose between:

• Quick Select (Grid)
• Mini Arcade (future lobby system)

GRID PAGE

Displays all available games.

Features:

• Card-based UI layout
• Video previews
• Game play counters
• Responsive design
• Gamepad navigation

GAME RUNTIME LAYER

Each game runs independently in its own folder.

Games now live inside the /games directory.

Example structure:

games/
  apple-catcher/
  blade-and-block/
  bird-duty/
  dodgeballs/
  mini-arcade/
  sumorai/
  x-pong/

Each game contains its own:

• index.html
• assets
• scripts
• sounds

Games are exported from TurboWarp Packager.

Each game remains fully self-contained.

UPDATED GAME IMPORT SYSTEM

Game updates are now automated using a build pipeline.

New workflow:

TurboWarp Packager
→ Export ZIP
→ Place ZIP in exports folder
→ Run build script
→ Import game
→ Patch game HTML
→ Commit and push changes

The process is largely automated.

INPUT ARCHITECTURE

There are three separate input systems working together.

SHARED MENU INPUT (ARCADE UI)

Handled by:

js/arcade-input.js

This script processes input for the menu system.

Supported inputs:

• Keyboard
• Gamepad

Supported actions:

left
right
up
down
select

These actions are used only for navigating the arcade UI.

IMPORTANT DESIGN DECISION:

The shared menu input system does NOT control gameplay.

It is strictly for navigating the arcade interface.

Used by:

• Index page
• Grid page

GAME INPUT SYSTEM

Each game handles its own input logic internally.

Games contain their own TurboWarp control logic.

Supported inputs inside games:

• Keyboard
• Touch controls
• Gamepad

Important design rule:

The arcade menu input system does not interfere with gameplay controls.

Games remain fully self-contained.

MOBILE CONTROLLER SYSTEM

Custom mobile controller script:

js/jay-mobile.js

This provides a virtual controller for mobile devices.

Features include:

• Virtual 8-direction D-pad
• Multi-touch support
• Face buttons (A, B, X, Y)
• Configurable control mapping
• Pointer-based controls
• Mobile fullscreen start
• Landscape orientation lock
• Scanline toggle button

CONTROL OVERRIDE SYSTEM

Overrides are centralized in:

scripts/control_overrides.py

Example:

GAME_OVERRIDES = {

  "blade-and-block": {
    "up": "j",
    "b": "c",
    "a": "w"
  },

  "apple-catcher": {
    "b": "ArrowLeft",
    "x": "ArrowRight"
  }

}

Overrides are injected automatically during patching.

This prevents needing to manually edit each game's HTML.

AUTOMATED PATCH SYSTEM

Python script:

scripts/patch_all_games.py

This script modifies TurboWarp exports automatically after importing.

Primary responsibilities:

MOBILE META PATCHING

Ensures mobile compatibility.

Adds:

viewport-fit=cover
apple-mobile-web-app-capable
mobile-web-app-capable
screen-orientation

SCRIPT INJECTION

Automatically inserts:

<script src="../../js/jay-mobile.js"></script>

and GoatCounter analytics configuration.

CONTROL OVERRIDE INJECTION

If a game appears in control_overrides.py the script automatically injects:

window.JAY_GAME_CONFIG

before jay-mobile.js.

SMART DETECTION

The patcher detects:

• existing viewport tags
• mismatched viewport formatting
• duplicate scripts
• existing overrides

PATCH REPORTING

The patcher prints detailed reports when run.

Example:

apple-catcher: PATCHED
  - viewport already correct
  - mobile meta tags already present
  - control overrides applied
  - jay-mobile.js already present
  - goatcounter already present
AUTOMATED BUILD SYSTEM

New script:

scripts/build_arcade.py

This script is now the primary workflow entry point.

Responsibilities:

• Scan exports folder for ZIP files
• Import each game into /games
• Replace old game folder contents
• Validate imports
• Run patch_all_games.py
• Optionally commit and push to GitHub
• Optionally delete processed ZIP files

Example command:

python scripts/build_arcade.py --push --clean-exports

This performs the entire update pipeline automatically.

EXPORT STAGING SYSTEM

TurboWarp packaged games are staged in:

exports/

Example:

exports/
  apple-catcher.zip
  blade-and-block.zip
  x-pong.zip

The build script imports these files automatically.

This staging system prevents accidental overwriting of live game folders.

CRT VISUAL SYSTEM

The arcade interface simulates CRT display behavior.

Features include:

• Scanlines overlay
• CRT vignette
• CRT power-on animation
• CRT collapse animation
• Shutdown flash line
• BIOS-style boot screen

The goal is to mimic the feel of a real arcade machine powering on.

NAVIGATION SYSTEM

Menu navigation supports:

Mouse
Keyboard
Gamepad
Mobile touch

All navigation ultimately triggers shared action events.

Game selection triggers CRT shutdown animation before loading.

GAME NAVIGATION RULES

Games include built-in options to return to:

• Grid page
• Mini Arcade

This ensures users always return to the arcade interface.
