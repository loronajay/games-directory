Project: Jay's Retro Arcade

Jay's Retro Arcade is a retro-themed arcade website that acts as the frontend OS for a future Raspberry Pi arcade cabinet.

The website handles:

• CRT boot visuals
• game selection grid
• keyboard/gamepad navigation
• mobile touch controls
• analytics tracking
• launching individual games

Games themselves are treated as independent apps.

Game Structure

Each game lives in:

games/<slug>/

Example:

games/
├ apple-catcher/
├ art-of-war/
├ bird-duty/
├ blade-and-block/
├ dodgeballs/
├ mini-arcade/
├ sumorai/
└ x-pong/

Minimum contents:

index.html
assets/
sounds/

Games are exported from TurboWarp Packager.

Game folders do not contain scripts folders.

Special Case
games/mini-arcade/

Exists but must NOT appear in the grid page.

The grid generator intentionally excludes it.

Build Pipeline

Primary script:

scripts/build_arcade.py

Workflow:

Export TurboWarp game as ZIP

Place ZIP into:

exports/

Run:

python scripts/build_arcade.py

The build script:

scan exports
import ZIPs into games/
preserve or create game.json
patch game HTML
regenerate grid
optional commit/push
optional clean exports

Typical deploy command:

python scripts/build_arcade.py --push --clean-exports
Metadata System

Each game can optionally include:

games/<slug>/game.json

Example:

{
  "title": "Blade & Block",
  "order": 20,
  "card_classes": ["desktop-only"]
}

Fields:

Field	Purpose
title	override auto title
order	manual grid ordering
card_classes	extra CSS classes
preview	override preview video name

If values are blank, defaults are used.

Starter file auto-created by build script:

{
  "title": "",
  "order": 9999,
  "card_classes": []
}
Grid Page
grid.html

Game cards are auto-generated.

Marker section:

<div class="game-grid">

<!-- AUTO-GENERATED-GAME-CARDS-START -->

<!-- AUTO-GENERATED-GAME-CARDS-END -->

</div>
Grid Generator

Script:

scripts/generate_grid.py

Responsibilities:

• scan games folder
• ignore mini-arcade
• read optional metadata
• auto-generate titles from slug
• build card HTML
• inject cards into grid.html

Preview naming convention:

previews/<slug>.mp4

If preview missing, card shows black video area.

Card HTML Structure

Cards must stay exactly like this because the JS depends on it:

<a class="game-card arcade-link" href="games/<slug>/index.html">

  <video muted loop preload="metadata">
    <source src="previews/<slug>.mp4" type="video/mp4">
  </video>

  <div class="game-title">Game Title</div>

  <div class="game-play-count">
    PLAYS: <span class="playCount">000000</span>
  </div>

</a>

CRT Visual System

UI simulates an arcade CRT.

Includes:

• scanlines
• vignette
• flicker
• BIOS boot sequence
• power-on animation
• CRT collapse shutdown animation

Analytics

GoatCounter tracks visits per page.

Example paths:

/games/apple-catcher/index.html
/games/x-pong/index.html

The grid page fetches counts and displays them on cards.
