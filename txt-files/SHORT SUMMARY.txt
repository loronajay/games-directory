Project name: Jay's Retro Arcade

This project is a retro-themed arcade gaming platform built as a website but designed to eventually run as a full arcade cabinet frontend.

The system is designed to support:

• Web hosted arcade
• Offline local arcade
• Raspberry Pi arcade cabinet
• Physical arcade controls

The website interface effectively functions as the "operating system" for the arcade.

CORE WEBSITE STRUCTURE

The site consists of two main UI pages:

index.html
Mode selection screen.

Players choose between:

• Quick Select (grid page)
• Mini Arcade (future lobby interface)

grid.html
Displays all games in a card-based grid.

Features:

• Preview videos
• Game play counters
• Responsive layout
• Keyboard/gamepad navigation

GAME STRUCTURE

Each game runs independently inside:

games/<game-name>/

Example:

games/apple-catcher/
games/blade-and-block/
games/x-pong/

Each game contains its own:

• index.html
• assets
• sounds
• scripts

Games are exported from TurboWarp Packager.

AUTOMATED BUILD PIPELINE

Game updates use a staging folder and automated build script.

Exports folder:

C:\Users\leoja\Desktop\Dad Games\full-games\exports

Workflow:

Export TurboWarp game as ZIP

Place ZIP in exports folder

Run build script

Game imports into /games directory

HTML patcher runs automatically

Changes optionally commit/push to GitHub

Primary build command:

python scripts/build_arcade.py --push --clean-exports

PATCH SYSTEM

Script:

scripts/patch_all_games.py

Automatically injects:

• Mobile viewport meta tags
• jay-mobile.js controller
• GoatCounter analytics
• Control overrides from control_overrides.py

INPUT SYSTEM

There are three separate input systems.

Menu Navigation

Handled by:

js/arcade-input.js

Supports:

• Keyboard
• Gamepad

Actions:

left
right
up
down
select

This is ONLY used for navigating the arcade interface.

Game Controls

Each game handles its own input internally through TurboWarp.

Mobile Controller

Script:

js/jay-mobile.js

Features:

• Virtual 8-direction D-pad
• Multi-touch buttons (A B X Y)
• Configurable key mapping
• Landscape orientation lock
• Fullscreen mobile start
• Scanline toggle

CONTROL OVERRIDE SYSTEM

Overrides are centralized in:

scripts/control_overrides.py

Example:

GAME_OVERRIDES = {

"blade-and-block": {
"up": "j",
"b": "c",
"a": "w"
},

"apple-catcher": {
"b": "ArrowLeft",
"x": "ArrowRight"
}

}

Overrides are automatically injected during patching.

CRT VISUAL SYSTEM

The UI simulates an arcade CRT display.

Features include:

• Scanlines
• CRT vignette
• Power-on animation
• Shutdown collapse animation
• Flash shutdown line
• BIOS-style boot screen

CURRENT PROJECT STATUS

The arcade currently includes:

• Multiple playable TurboWarp games
• Fully working arcade interface
• Gamepad + keyboard navigation
• Mobile touch controller
• Play count analytics
• Automated build + patch system
• Centralized control override management

The project represents the first functional version of the arcade platform frontend.

LONG TERM GOAL

The system is intended to eventually run as a full arcade cabinet frontend on Raspberry Pi with physical controls.

The website version is the first implementation of that system.
