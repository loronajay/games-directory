JAY RETRO ARCADE — BUILD AND DEPLOY WORKFLOW

PROJECT OVERVIEW

The arcade project now uses a unified automated workflow built around a single build script.

The intended update process is:

Export updated games from TurboWarp Packager as ZIP files

Place the ZIP files into the exports staging folder

Run the arcade build script

The build script imports games, patches them, and optionally pushes changes

Preview video creation remains a manual step for each game.



MAIN BUILD SCRIPT

scripts/build_arcade.py

This script performs the full arcade update process.

Responsibilities:

• Scan the exports folder for ZIP files
• Import each ZIP into games/<game-name>/
• Clear old game contents before replacing
• Validate imported game structure
• Run the patcher automatically
• Optionally commit and push changes
• Optionally delete processed ZIPs

The build script is now the main command used to update the arcade.

PATCH SCRIPT

scripts/patch_all_games.py

This script modifies game HTML files automatically.

Responsibilities:

• Inject correct mobile viewport meta tags
• Inject jay-mobile.js controller script
• Upgrade old mobile controller paths
• Inject GoatCounter analytics block
• Inject control overrides if configured
• Optionally commit and push changes

CONTROL OVERRIDES

scripts/control_overrides.py

This file defines custom button mappings for specific games.

Example:

GAME_OVERRIDES = {

"blade-and-block": {
"up": "j",
"b": "c",
"a": "w"
},

"apple-catcher": {
"b": "ArrowLeft",
"x": "ArrowRight"
}

}

These overrides are automatically injected into each game by the patcher.

PATCHER FEATURES

Mobile compatibility

The patcher ensures every game HTML includes:

viewport meta tag
apple-mobile-web-app-capable
mobile-web-app-capable
screen-orientation landscape

Mobile controller injection

Every game automatically receives:

jay-mobile.js

Legacy controller paths are upgraded automatically.

Analytics injection

The patcher injects GoatCounter tracking using:

window.goatcounter path tracking

Control overrides

If a game appears in control_overrides.py the patcher injects:

window.JAY_GAME_CONFIG

before the mobile controller script.

BUILD SCRIPT EXECUTION ORDER

The build script runs the following steps:

Scan exports folder for ZIP files

Import ZIP contents into games/<game-name>/

Replace existing game folder contents

Validate import success

Run patch_all_games.py

Optionally commit and push changes

Optionally delete processed ZIPs

If any import fails, patching stops and the build aborts.
