GAME STRUCTURE

Each game lives in:

games/<slug>/

games/

Game folders must not contain scripts folders.

SPECIAL CASE

The folder:

games/mini-arcade/

exists but must never appear in the grid page.

The grid generator intentionally excludes it.

BUILD PIPELINE

Primary script:

scripts/build_arcade.py

Workflow:

Export TurboWarp game as ZIP

Place ZIP inside:

exports/

Run:

python scripts/build_arcade.py

The build script automatically:

• scans exports
• imports ZIPs into games/
• preserves or creates game.json
• patches game HTML
• regenerates the grid page
• optionally commits/pushes to Git
• optionally cleans exports

Typical deploy command:

python scripts/build_arcade.py --push --clean-exports

This script is the source of truth for the entire arcade system.

Future solutions should integrate with this pipeline.

GRID SYSTEM

grid.html contains auto-generated game cards.

Marker section:

<div class="game-grid"> <!-- AUTO-GENERATED-GAME-CARDS-START --> <!-- AUTO-GENERATED-GAME-CARDS-END --> </div>

Cards are generated by:

scripts/generate_grid.py

Responsibilities:

• scan games folder
• ignore mini-arcade
• read optional metadata
• auto-generate titles from slug
• build card HTML
• inject cards into grid.html

Manual editing inside this section will be overwritten.

CARD HTML STRUCTURE (MUST NOT CHANGE)

The grid page JavaScript depends on this exact structure:

<a class="game-card arcade-link" href="games/<slug>/index.html"> <video muted loop preload="metadata"> <source src="previews/<slug>.mp4" type="video/mp4"> </video> <div class="game-title">Game Title</div> <div class="game-play-count"> PLAYS: <span class="playCount">000000</span> </div> </a>

Class names and structure must remain unchanged unless explicitly requested.

PREVIEW SYSTEM

Preview videos follow the naming convention:

previews/<slug>.mp4

Examples:

previews/apple-catcher.mp4
previews/x-pong.mp4

If the preview video is missing, the card should still render with a black video area.

METADATA SYSTEM

Each game may optionally include:

games/<slug>/game.json

Example:

{
"title": "Blade & Block",
"order": 20,
"card_classes": ["desktop-only"]
}

Fields:

title — override auto title
order — manual grid ordering
card_classes — extra CSS classes
preview — override preview video name

If values are blank, defaults are used.

Starter metadata is auto-generated:

{
"title": "",
"order": 9999,
"card_classes": []
}

Manual metadata editing should not be required for normal games.

PATCH SYSTEM

Script:

scripts/patch_all_games.py

Automatically injects platform features into each game.

Injected features include:

• mobile viewport meta tags
• jay-mobile.js controller
• GoatCounter analytics
• control overrides

Game HTML files should not be manually edited.

All platform functionality should be injected through the patch system.
