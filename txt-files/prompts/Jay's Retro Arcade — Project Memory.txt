Jay's Retro Arcade — Project Memory Prompt

You are assisting with the development of Jay’s Retro Arcade, a retro-themed arcade platform that functions as a browser-based arcade operating system and the future frontend for a Raspberry Pi-powered arcade cabinet ecosystem.

The project is being developed by a solo developer, and the system is intentionally designed for automation, scalability, and minimal manual maintenance.

Current Project Status

Jay’s Retro Arcade currently exists as a fully functional arcade website platform that supports multiple input methods and launches independent games.

The platform already supports:

• desktop keyboard controls
• desktop gamepad controls
• mobile touch controls
• controller-agnostic input design for future arcade hardware

The website functions as the main arcade operating system interface, which will later also power physical arcade cabinets.

Current Game Library

Games currently present on the platform include:

• Apple Catcher
• Art of War
• Bird Duty
• Blade & Block
• Dodgeballs
• Mini Arcade (special case — hidden from grid)
• Sumorai
• X-Pong

These games are exported using TurboWarp Packager and treated as independent game applications.

Each game lives in:

games/<slug>/

Games include their own:

• assets
• sounds
• internal high score systems.

Platform-level features are injected automatically through the patch system.

Current Platform Features

The arcade platform currently includes:

• CRT-style boot sequence
• arcade game selection grid
• keyboard navigation
• gamepad navigation
• mobile controller system
• game preview videos on the grid
• GoatCounter analytics integration
• automated game ingestion pipeline

The platform is designed to behave like a retro arcade operating system.

Target Near-Term Milestone

The current primary development goal is:

15 polished arcade games

Each game should be:

• stable

The focus is on arcade-style gameplay designed for high score competition.

• original arcade games

• arcade cabinet hardware
• competitive leaderboards
• tournaments

Players will eventually interact with the platform through:

Design Philosophy

The system prioritizes:

• automated build pipelines
• automated grid generation
• automated HTML patching
• independent game folders
• scalable game ingestion
• minimal manual editing

The platform is intentionally designed to scale into a large arcade library with minimal maintenance overhead.
