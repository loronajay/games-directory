Jay's Retro Arcade — Quick Context Prompt

You are helping develop Jay’s Retro Arcade, a retro-themed arcade platform that acts as both:

• a browser-based arcade operating system
• the frontend for a future Raspberry Pi arcade cabinet ecosystem

The system launches independent TurboWarp-exported games stored in games/<slug>/.
Games must remain isolated and should not require manual editing.

The platform automatically manages content using a build pipeline:

scripts/build_arcade.py

This pipeline handles:

• importing TurboWarp ZIP exports from exports/
• patching game HTML via scripts/patch_all_games.py
• regenerating the game grid via scripts/generate_grid.py

Game cards in grid.html are auto-generated and must not be manually edited.
Their HTML structure must remain unchanged because existing JavaScript depends on it.

The project prioritizes:

• automation
• scalable game ingestion
• minimal manual editing
• controller-agnostic input (keyboard, gamepad, mobile touch, future arcade joystick)

The platform currently runs in a web environment, but the long-term goal is a Raspberry Pi-powered arcade cabinet OS.

The arcade ecosystem will eventually include:

• ~15+ original arcade games
• global leaderboards
• player profiles
• tournaments
• custom arcade cabinet hardware
• a portable backpack arcade system

When proposing solutions, respect the existing architecture and build pipeline.
